# Issues

Numbered enhancement / follow-up notes captured outside of git issues. One file per item, prefixed with a zero-padded sequence number. Newest at the bottom — pick the next free number on creation.

Format per file: a one-liner front-matter + a problem statement + a sketch / decision-criteria / acceptance criteria. Optional. Keep concise.

Resolve by either:
- Closing it in the file body with a `## Resolution` section + the commit hash, then moving the file to `docs/issues/closed/`, or
- Deleting if it turned out to be wrong / superseded.
