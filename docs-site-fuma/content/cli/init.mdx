## Overview

The `init` command creates a new agentic-primitives repository with the standard directory structure and configuration files.

## Usage

```bash
agentic-p init [OPTIONS] [PATH]
```

## Arguments

| Argument | Description | Required |
|----------|-------------|----------|
| `[PATH]` | Directory to initialize | No (defaults to current directory) |

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `-f, --force` | Overwrite existing files | `false` |
| `--template <NAME>` | Use a template (`minimal`, `standard`, `full`) | `standard` |
| `--no-examples` | Skip creating example primitives | `false` |
| `-v, --verbose` | Show detailed output | `false` |

## Examples

### Basic Usage

```bash
# Initialize in current directory
agentic-p init

# Initialize in a specific directory
agentic-p init my-primitives

# Initialize with full template (includes examples)
agentic-p init --template full
```

### With Options

```bash
# Force overwrite existing files
agentic-p init --force

# Minimal setup (no examples, minimal config)
agentic-p init --template minimal --no-examples
```

## Created Structure

The `standard` template creates:

```
my-primitives/
├── primitives/
│   └── v1/
│       ├── agents/
│       ├── commands/
│       │   └── meta/
│       ├── skills/
│       ├── tools/
│       └── hooks/
├── providers/
│   ├── models/
│   │   └── anthropic/
│   └── agents/
│       └── claude-code/
├── specs/
│   └── v1/
│       ├── prompt-meta.schema.json
│       ├── tool-meta.schema.json
│       └── hook-meta.schema.json
├── primitives.config.yaml
└── .gitignore
```

## Configuration File

The created `primitives.config.yaml`:

```yaml
# agentic-primitives configuration
spec_version: "v1"

# Repository metadata
repository:
  name: "my-primitives"
  description: "My agentic primitives collection"

# Default settings
defaults:
  model:
    provider: anthropic
    name: claude-4-sonnet

# Build configuration
build:
  output_dir: "build"
  providers:
    - claude

# Validation settings
validation:
  strict: true
  check_hashes: true
```

## Templates

| Template | Description |
|----------|-------------|
| `minimal` | Bare directory structure, no examples |
| `standard` | Standard structure with base configuration |
| `full` | Complete setup with example primitives |
