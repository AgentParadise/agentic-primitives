Before you begin, ensure you have:

## Step 2: Initialize Your Repository

```bash
# Create a new directory for your primitives
mkdir my-primitives && cd my-primitives

# Initialize agentic-p repository
agentic-p init
```

This creates:

```
my-primitives/
├── primitives/
│   └── v1/
│       ├── agents/
│       ├── commands/
│       ├── skills/
│       ├── tools/
│       └── hooks/
├── providers/
├── specs/
│   └── v1/
└── primitives.config.yaml
```

## Step 3: Create Your First Primitive

Let's create a Python expert agent:

```bash
agentic-p new agent python/python-pro
```

This creates a new agent primitive:

```
primitives/v1/agents/python/python-pro/
├── python-pro.meta.yaml    # Metadata and version registry
└── python-pro.v1.md        # Prompt content (version 1)
```

<Accordions>
  <Accordion title="View the generated files">
    **python-pro.meta.yaml:**
    ```yaml
    spec_version: "v1"
    type: agent
    name: python-pro
    description: "Python expert agent"

    versions:
      - version: 1
        status: active
        hash: blake3:pending
        created: "2025-12-09"
        notes: "Initial version"

    default_version: 1

    model:
      provider: anthropic
      name: claude-4-sonnet
    ```

    **python-pro.v1.md:**
    ```markdown
    # Python Pro Agent

    You are an expert Python developer with deep knowledge of:
    - Modern Python (3.11+) best practices
    - Type hints and static analysis
    - Testing with pytest
    - Async/await patterns
    - Package management with UV

    ## Guidelines

    1. Always use type hints
    2. Follow PEP 8 style guidelines
    3. Write comprehensive docstrings
    4. Suggest tests for new functionality
    ```
  </Accordion>
</Accordions>

## Step 4: Validate Your Primitives

```bash
# Validate all primitives
agentic-p validate

# Inspect a specific primitive
agentic-p inspect python/python-pro
```

<Callout type="info">
Validation runs three layers of checks:
1. **Structural** - File organization and naming
2. **Schema** - YAML/JSON schema compliance
3. **Semantic** - Cross-references and version integrity
</Callout>

## Step 5: Build for Your Provider

```bash
# Build for Claude Code
agentic-p build --provider claude
```

This generates provider-specific output in `build/claude/`.

## Step 6: Install to Your Project

```bash
# Install to your project's .claude/ directory
agentic-p install --provider claude --project

# Or install globally
agentic-p install --provider claude --global
```

<Callout type="info" title="Congratulations!">
You've created, validated, and deployed your first agentic primitive.
</Callout>

## Common Commands

| Command | Description |
|---------|-------------|
| `agentic-p init` | Initialize a new repository |
| `agentic-p new <type> <path>` | Create a new primitive |
| `agentic-p validate` | Validate all primitives |
| `agentic-p list` | List all primitives |
| `agentic-p build --provider <name>` | Build for a provider |
| `agentic-p install --provider <name>` | Install to project |
