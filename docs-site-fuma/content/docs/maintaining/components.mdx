## Overview

Fumadocs provides a rich set of MDX components for creating interactive documentation. This page covers the most commonly used components.

<Callout type="info">
For the full component reference, see [Fumadocs Components](https://fumadocs.vercel.app/docs/ui/components).
</Callout>


## Accordions

### Accordion Group

```markdown
import { Accordions, Accordion } from 'fumadocs-ui/components/accordion';

<Accordions>
  <Accordion title="What is agentic-p?">
    agentic-p is a CLI tool for managing agentic primitives.
  </Accordion>
  <Accordion title="How do I install it?">
    Run cargo install agentic-p or brew install agentic-p.
  </Accordion>
</Accordions>
```

<Accordions>
  <Accordion title="What is agentic-p?">
    agentic-p is a CLI tool for managing agentic primitives.
  </Accordion>
  <Accordion title="How do I install it?">
    Run `cargo install agentic-p` or `brew install agentic-p`.
  </Accordion>

## Steps

### Sequential Steps

```markdown
import { Step, Steps } from 'fumadocs-ui/components/steps';

<Steps>
  <Step>
    ### Install the CLI
    Run cargo install agentic-p
  </Step>
  <Step>
    ### Initialize your project
    Run agentic-p init in your project root
  </Step>
  <Step>
    ### Create your first primitive
    Run agentic-p new agent my-agent
  </Step>
</Steps>
```

<Steps>
  <Step>
    ### Install the CLI
    Run `cargo install agentic-p`
  </Step>
  <Step>
    ### Initialize your project
    Run `agentic-p init` in your project root
  </Step>
  <Step>
    ### Create your first primitive
    Run `agentic-p new agent my-agent`
  </Step>
</Steps>

## Importing Components

Remember to import components at the top of your MDX file:

```markdown
import { Cards, Card } from 'fumadocs-ui/components/card';
import { Tabs, Tab } from 'fumadocs-ui/components/tabs';
import { Accordions, Accordion } from 'fumadocs-ui/components/accordion';
import { Step, Steps } from 'fumadocs-ui/components/steps';
import { Callout } from 'fumadocs-ui/components/callout';
```

## Component Checklist

When writing docs, consider using:

- [ ] **Callouts** for important information
- [ ] **Cards** for navigation and feature highlights
- [ ] **Tabs** for platform-specific instructions
- [ ] **Steps** for tutorials and guides
- [ ] **Accordions** for FAQs or optional details
- [ ] **Tables** for reference data
