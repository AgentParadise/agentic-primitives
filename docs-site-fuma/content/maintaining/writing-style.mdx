## Voice & Tone

### Be Direct and Practical

<Tabs items={["✅ Do", "❌ Don't"]}>
  <Tab value="✅ Do">
    ```markdown
    Run `agentic-p validate` to check your primitives.
    ```
  </Tab>
  <Tab value="❌ Don't">
    ```markdown
    You might want to consider running the validate command
    which could potentially help you check your primitives.
    ```
  </Tab>
</Tabs>

### Use Active Voice

<Tabs items={["✅ Do", "❌ Don't"]}>
  <Tab value="✅ Do">
    ```markdown
    The CLI validates all primitives in the repository.
    ```
  </Tab>
  <Tab value="❌ Don't">
    ```markdown
    All primitives in the repository are validated by the CLI.
    ```
  </Tab>
</Tabs>

### Address the Reader as "You"

<Tabs items={["✅ Do", "❌ Don't"]}>
  <Tab value="✅ Do">
    ```markdown
    You can create a new agent with the `new` command.
    ```
  </Tab>
  <Tab value="❌ Don't">
    ```markdown
    Users can create a new agent with the `new` command.
    ```
  </Tab>
</Tabs>

## Page Structure

### Frontmatter

Every page must have:

```yaml
---
title: 'Page Title'
description: 'A brief description for SEO and previews'
---
```

### Recommended Sections

1. **Overview** - Brief intro (2-3 sentences)
2. **Prerequisites** (if applicable)
3. **Main Content** - Steps, concepts, or reference
4. **Examples** - Real-world usage
5. **Next Steps** - Link to related pages

---

## Formatting Standards

### Headings

- Use `##` for main sections (H2)
- Use `###` for subsections (H3)
- Avoid H4+ when possible (indicates need to restructure)
- Use sentence case: "Getting started" not "Getting Started"

### Code Blocks

Always specify the language:

````markdown
```bash
agentic-p validate
```

```yaml
name: my-agent
type: agent
```

```python
def validate_primitive(path: str) -> bool:
    return True
```
````

### Inline Code

Use backticks for:
- Commands: `agentic-p build`
- File names: `agent.meta.yaml`
- CLI flags: `--strict`
- Values: `true`, `false`, `null`

### Lists

Use bullet points for unordered items:
```markdown
- First item
- Second item
- Third item
```

Use numbered lists for sequential steps:
```markdown
1. First step
2. Second step
3. Third step
```
