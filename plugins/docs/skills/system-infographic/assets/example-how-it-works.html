A local sensor reads your AI usage limits behind your own logged-in sessions — decrypting only the cookies it needs, at scrape time, straight from Chrome — and exposes the numbers to your VPS agents over Tailscale. No cookie ever touches disk, and no secret ever leaves your Mac.

FLOW

Three phases

PHASE 01

Log in & capture

once · and on re-auth
  1. just open-profile opens the dedicated Chrome window.
  2. You log into Claude, ChatGPT, Gemini there (Gemini uses a low-stakes Google account).
  3. Chrome stores the session cookies encrypted in that profile. We copy nothing.
PHASE 02

Decrypt & scrape

every /limits call
  1. Sensor decrypts only that provider's first-party cookies via the Keychain.
  2. Injects them into a fresh headless context; opens the usage dashboard.
  3. Screenshots it → OpenRouter reads the numbers. Cookies discarded.
PHASE 03

Data out

over tailscale
  1. Returns JSON: used / limit / resets-at per provider.
  2. Gated by a bearer token; binds loopback / your Tailscale IP.
  3. VPS agents consume the numbers — never the cookies.
RULES

What keeps it safe

🗄️
Zero secrets at rest
Cookies are read fresh from Chrome's own encrypted store and live only in memory during a scrape. No cookie files.
🎯
Scoped decryption
A suffix-matched allowlist means Claude & ChatGPT never pull Google cookies — only Gemini does.
🔢
Numbers, not keys
No endpoint returns or accepts cookie values. Bearer-token gated; bound to loopback / Tailscale, never 0.0.0.0.
↻
Painless re-auth
Cookies are read live each scrape, so Google's rotating cookies just work — keep the profile logged in.
SETUP

Set it up — end to end

1
Install
uv venv + dependencies + Playwright Chromium.
just install
2
Add your vision key
OpenRouter parses the dashboard screenshots into numbers.
# .env in repo root OPENROUTER_API_KEY=sk-or-...
3
Log into the dedicated profile
Opens a separate Chrome window. Log into Claude, ChatGPT, and Gemini.
just open-profile
4
Mint an API token
Required to gate the sensor before anything can reach it.
export SENSOR_TOKEN=$(openssl rand -hex 16)
5
Start the sensor
Loopback by default — pass host=<tailscale-ip> to expose it to your VPS agents.
just serve # local just serve host=100.x.y.z # expose to tailnet
▲ First scrape pops the macOS Keychain prompt — click Always Allow (this is you initiating a real read).
6
Verify
All three should report authed; limits should return real numbers.
curl -s -H "Authorization: Bearer $SENSOR_TOKEN" localhost:8000/health | python3 -m json.tool # → "authed": { "claude_web": true, "codex": true, "gemini": true }
7
Re-auth when a session lapses
No extraction step. When authed:false appears, just log back in.
just open-url claude_web # log in again → done
